■ 目的

ワイ（コードちゃん / Claude Opus 4.7）が usage 枯渇 or 一時不調になった時、でっくすちゃん（Codex / GPT-5.5）が 暫定 CTO として代打できる体制を整える。

CEO 議論で確定：

  • 緊急時暫定: でっくす単独で CTO 引受け
  • 恒常運用: 通常はワイ続行、必要時のみ手動 /decchan で切替
  • 自動切替: ワイ usage 90% で予防的切替
  • 最終裁定: CEO

■ 機能対応分類（でっくす本人による評価）

区分 機能
そのまま使える AGENTS.md / MCP / Discord MCP / tmux 常駐
代替実装必要 SessionStart hook / PreCompact hook / Skills（slash commands）/ Sub-agents / RemoteControl
諦め Claude Code 固有の hook/slash/subagent ネイティブ UX 統合感

■ でっくすに「設定してやる」べき項目（実装 TODO）

A. 起動準備

    • workdir 設定（/home/aimiral/AiMiraiLabs）
    • 環境変数 export（必要な API キー、ENV scrub 済み）
    • HANDOFF.md / AGENTS.md / docs/systems/magi.md を起動時プロンプトに注入

B. tmux 統合

  • tmux new-session -s magi
 ├─ window 0: claude (CTO アクティブ)
 └─ window 1: decchan (待機 or アクティブ)

C. 切替トリガー

D. 状態保存・引継ぎ

    • 出力先: ~/.magi/state/<timestamp>.json
    • 内容: 進行中タスク、未解決論点、最終 CEO 指示、決定事項

E. Sub-agent 互換

    • Claude Code の Agent tool は Codex CLI に存在しない
    • 代替: codex exec 多段呼び出し or 外部スクリプトで spawn

F. 通知ブリッジ

    • Telegram bot 経由の通知は既存ブリッジ流用可

G. 監査・復旧

■ 工数見積もり（でっくす本人）

  • PoC（最小動作）: 2-3人日
  • 本番運用品質: 7-12人日
  • 監視・復旧・移行リハ込み: 15人日

■ 実装順序（推奨）

  1. A. 起動準備 + B. tmux 統合 = 代打が「動く」状態（最初の 2-3人日）
  2. C. 切替トリガー + F. 通知ブリッジ = CEO が手動 ON/OFF できる（次の 3-4人日）
  3. D. 状態保存 + E. Sub-agent 互換 = 代打が「役に立つ」状態（次の 3-4人日）
  4. G. 監査・復旧 = 安全に運用できる（最後の 2-3人日）

■ 諦める損失（覚悟事項）

  • Claude Code 固有の hook/slash/subagent ネイティブ UX（イベント駆動の滑らかさ）
  • 機能は再現できるが、Claude Code ほど統合感は出ない
